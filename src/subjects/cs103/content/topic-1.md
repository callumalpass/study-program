## Topic Map

Classes and objects introduce the move from functions operating on loose data to objects that own state and behavior. Use this topic to learn the syntax, then keep asking whether a class represents a meaningful responsibility.

## Learning Objectives

- Define classes and create independent object instances
- Use `__init__`, `self`, instance attributes, and class attributes correctly
- Add methods that preserve object invariants
- Implement basic special methods such as `__repr__`, `__str__`, and `__eq__`
- Recognize when a simple function or data structure is better than a new class

## Design Focus

By the end of this topic, a class should not feel like a container for random fields. It should have a reason to exist: state it owns, behavior it provides, and rules it protects.
