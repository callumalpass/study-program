## Topic Map

Design principles and testing bring the course together. The goal is maintainable object-oriented code: clear responsibilities, explicit dependencies, substitutable objects, and tests that allow refactoring.

## Learning Objectives

- Choose composition or inheritance based on relationship and change pressure
- Use SOLID as a review checklist, not as ritual architecture
- Inject dependencies so collaborators can be replaced in tests
- Test public behavior, invariants, and object collaboration
- Refactor toward smaller interfaces without changing external behavior

## Design Focus

Good OOP code is easy to change because its dependencies are visible and its behavior is verified. Prefer small, testable designs over large architectures that only look sophisticated.
