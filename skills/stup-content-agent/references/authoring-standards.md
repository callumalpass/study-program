# Authoring Standards

Prefer additive, stable changes.

ID rules:

- Do not rename IDs casually.
- Do not reuse deleted IDs for new meanings.
- Increment activity version when semantics change.
- Create a new activity ID when an assessment is substantially different.
- Keep question IDs stable for wording fixes.

Activity quality:

- Every quiz answer needs an explanation.
- Coding exercises need starter code, solution, and tests unless explicitly
  non-automated.
- Written exercises need a model answer or rubric.
- Projects need requirements, milestones, and rubric criteria.
- Flashcards must test one fact or distinction.

Overlay behavior:

- Personalized content belongs in the configured authoring pack.
- Upstream pack overrides must be explicit and reviewable.
- Prefer remediation modules over destructive rewrites of shared subjects.
